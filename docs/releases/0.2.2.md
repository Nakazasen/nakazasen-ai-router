# Nakazasen AI Router 0.2.2

## Highlights

- Provider key onboarding docs for Gemini, Groq, and OpenRouter.
- Safe `provider_keys.example.env` template with empty values only.
- Local CLI key setup helper with masked input and repo-internal write protection.
- Static local-only key setup helper for copy/download workflows.
- Install, update, and uninstall documentation.
- Local path leak guard for machine-specific paths and key file names.

## Install or update

```powershell
py -m pip install --upgrade --force-reinstall "nakazasen-ai-router @ git+https://github.com/Nakazasen/nakazasen-ai-router.git@v0.2.2"
```

## Safety notes

- No API keys are bundled in the repository.
- At the time of this release, key files were recommended outside the repository. As of 0.2.3, the explicitly ignored root `API Key.txt` local-development convention is also supported for live smoke and discovery scripts.
- The static key setup page is local-only and does not call providers.
- Browser-to-provider live calls remain out of scope.

## Verification

Release verification includes local path audit, provider key onboarding tests, docs/mojibake audits, full offline tests, package build, and install-from-wheel smoke.
