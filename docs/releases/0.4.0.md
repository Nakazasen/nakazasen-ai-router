# Nakazasen AI Router 0.4.0

Released: 2026-07-23

## Highlights

- Added opt-in installed-version awareness and a safe CLI: `version`, `update --check`, and explicit `update --apply`.
- Update application previews the exact current-interpreter pip command and requires confirmation unless controlled automation passes `--yes`.
- Added an auditable free-tier schema and built-in provider catalog with source/date/confidence provenance and shared-pool deduplication.
- Added `free-tiers` human/JSON reporting and a free-tier preference signal for `cheap` and `quota` routing modes.
- Added process-local period usage estimation clearly labeled `estimated_local`; provider dashboards remain authoritative.

## Honest free-token positioning

The built-in catalog does not invent monthly token grants from dynamic rate limits. Its current audited recurring numeric total is therefore **0 tokens/month**. Dynamic/unlimited plans are listed separately and can remain routing-eligible when their provenance is current. Nakazasen does not claim OmniRoute's `1.53B free tokens/month` without a current, reproducible Nakazasen-specific catalog.

## Compatibility and safety

- Existing positional `ScoreWeights` and `RoutingScore` construction remains compatible because the new field is appended.
- Hard policy, privacy, cooldown, quota, and budget checks still win over free-tier preferences.
- Importing, constructing, or routing does not perform update network access or environment mutation.
- Raw keys, Authorization headers, prompts, and provider payloads are not included in free-tier/update metadata.

## Verification

The release gate includes the offline suite, compilation and repository audits, release consistency, clean sdist/wheel build, isolated wheel install/CLI smoke, and a bounded opt-in live provider smoke using an ignored local key file.
