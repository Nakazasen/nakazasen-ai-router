# Nakazasen AI Router 0.3.0

## Highlights

- Added explainable weighted auto-routing with `balanced`, `fast`, `cheap`, `quality`, and `quota` mode packs plus custom score weights.
- Added thread-safe, process-local shared quota pools and named fixed windows with request/token limits, reservations, headroom, and token reconciliation.
- Added normalized provider token usage, catalog provenance, and conservative cost estimates that remain `unknown` when verified pricing or usage splits are unavailable.
- Integrated quota and accounting into both synchronous and asynchronous router lifecycles without weakening budget, privacy, provider, cooldown, or last-resort hard constraints.
- Added standardized bilingual release/packaging runbooks, release consistency automation, and a human/AI architecture handoff map with Mermaid diagrams.

## Install or update

```powershell
py -m pip install --upgrade --force-reinstall "nakazasen-ai-router @ git+https://github.com/Nakazasen/nakazasen-ai-router.git@v0.3.0"
```

## Compatibility and limitations

- Existing positional `ModelCapability` and `ProviderQuotaProfile` construction remains compatible because new fields are appended and defaulted.
- Existing static catalog entries do not invent provider prices; their cost status remains `unknown` until pricing is explicitly sourced and verified.
- Shared quota pools and flexible windows are process-local. They are not distributed or multi-process coordination.
- Flexible windows are fixed windows, not rolling/sliding windows.
- Profiles sharing one `pool_id` should use compatible policies.

## Safety notes

- Raw API keys, Authorization headers, prompts, and raw provider responses are excluded from route attempts, accounting metadata, state, and quota snapshots.
- `API Key.txt` remains a local development-only ignored file and is never packaged or committed.
- Live network calls remain explicit opt-in.

## Verification

Release verification includes full offline regression tests, compile and repository audits, release metadata consistency, clean sdist/wheel build, isolated wheel installation, and a bounded sanitized live-provider smoke test.
