# Nakazasen AI Router 0.2.3

## Highlights

- Added opt-in startup catalog refresh for Gemini and OpenAI-compatible provider model endpoints.
- Added safe merging of discovered chat-capable models ahead of static fallbacks, without persisting catalog changes.
- Refreshed Gemini and DeepSeek built-in model identifiers and aliases.
- Added local developer default key file `API Key.txt`, protected by an exact Git ignore rule.
- Expanded API-key integration guidance for host applications using environment variables, private `env` mappings, or secret managers.

## Install or update

```powershell
py -m pip install --upgrade --force-reinstall "nakazasen-ai-router @ git+https://github.com/Nakazasen/nakazasen-ai-router.git@v0.2.3"
```

## Safety notes

- No API keys are bundled or pushed.
- `API Key.txt` is a local development-only ignored file; never force-add it.
- Startup refresh requires both `enable_network=True` and `refresh_models_on_startup=True`.
- Discovery failures retain static provider model defaults and do not block router startup.

## Verification

Release verification includes focused and full offline tests, compile checks, local-path audit, Git ignore validation, package build, and install-from-wheel smoke.
